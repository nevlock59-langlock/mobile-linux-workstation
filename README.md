# Mobile Linux Workstation

A recovery log and reproducible setup note for my Android-based mobile Linux workstation.

This repository documents:
- Termux CLI environment
- Python / Node runtime
- Xfce4 desktop environment
- VNC-related scripts
- Mobile AI coding workflow
- GitHub + cloud deploy loop

The goal is not to build the smallest possible screen.

The goal is to build the smallest useful end-to-end development flow.

## Why this repository exists

This is not a polished tutorial yet.

I originally built and used this environment years ago, then partially forgot the setup details.

This repository is an attempt to recover, preserve, and document:
- runtime artifacts
- desktop environment traces
- shell scripts
- mobile development workflow ideas

## Current confirmed components

- Android phone
- Termux
- Python runtime
- Node.js / npm runtime
- Git / GitHub workflow
- VNC-related scripts
- Xfce4 configuration backup
- Mobile-first AI coding workflow

## Interesting findings during recovery

During the recovery process, I rediscovered:
- old VNC startup scripts
- Xfce4 panel and session configuration
- desktop environment traces
- shell customization files
- runtime setup artifacts

Some parts of the environment survived for years inside the same phone.

## Philosophy

Instead of treating the phone as a weak desktop replacement, this setup evolved toward:

- lightweight local orchestration
- CLI-driven workflow
- AI-assisted coding loop
- Git-based deployment
- cloud-assisted runtime workflow
